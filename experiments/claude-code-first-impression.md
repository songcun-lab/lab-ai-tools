# Claude Code · First Impression

## 使用场景

我是在尝试 **CLI 场景下的 AI 编程与配置理解** 时，决定测试 Claude Code 的。

具体背景包括：

- 使用过 GitHub Copilot，但更偏 IDE 内补全
- 尝试过 Gemini / GPT，对话能力强，但 CLI 连贯性一般
- 希望找到一种 **“在终端里持续协作”的方式**
- 实际任务包括：  
  - CLI 工具安装与报错排查  
  - 配置文件理解（YAML / JSON）  
  - 将模糊需求逐步收敛为可执行命令  

---

## 实际过程

整体体验分为三个阶段：

**1️⃣ 启动与接入**
- 安装过程本身不复杂，但对环境依赖（Node / PATH）较敏感
- 第一次启动时，对“它到底能做什么”并不直观

**2️⃣ 实际使用**
- 在连续命令、上下文保持方面，明显优于“单轮对话式 GPT”
- 对 CLI 输出的理解能力较强，能基于错误信息给出下一步建议
- 但在某些情况下，**会显得过于自信**，需要人工校验

**3️⃣ 心理变化**
- 一开始是“尝试心态”
- 中途出现“这东西有点意思”
- 目前处在“值得继续投入，但不盲信”的状态

---

## 对比感受

**对比对象：**
- GitHub Copilot
- Gemini CLI
- ChatGPT（网页 / API）

**我的主观判断：**

- Claude Code 更像一个 **“终端里的协作者”**
- Copilot 更像 **“代码补全工具”**
- GPT / Gemini 更适合 **策略讨论与解释**

Claude Code 的优势在于：
- 上下文连续
- 对 CLI 真实输出更敏感
- 更像“一起做事”，而不是“问一句答一句”

不足之处在于：
- 仍需要人工判断，不适合全自动
- 对新手来说，学习成本略高
